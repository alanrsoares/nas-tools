# Product

## Register

product

## Users

Solo home user managing a personal NAS. Technically proficient: runs self-hosted services, knows what Plex is, understands the difference between an audiobook and a podcast. Uses this tool sporadically after a batch download completes. Primary task: scan the staging directory, review the move plan, fix any artist names, execute. No onboarding needed. No patience for noise.

## Product Purpose

Staging console for Plex media library management. Scans a downloads directory, categorizes media (music, movies, TV, audiobooks), builds a move plan with inferred destinations, lets the user correct errors, and executes the move. Success means zero cognitive overhead between download completion and Plex availability.

## Brand Personality

Focused, precise, unobtrusive. Raycast energy: fast to scan, dense with intent, never decorative. The interface should feel like a sharp tool, not a product trying to sell itself.

## Anti-references

- Generic SaaS: purple gradients, floating CTAs, onboarding checklists, empty-state illustrations with cartoon characters.
- Bootstrap admin panels: heavy blue sidebars, overweight navbars, table borders everywhere.
- Hacker terminal aesthetic: green-on-black, monospace everything, for-show darkness.

## Design Principles

1. Serve the scan-review-execute workflow. Every element either aids that loop or shouldn't exist.
2. Expert user by default. No tooltips explaining obvious things, no confirmation dialogs for low-stakes actions.
3. Information density over decoration. More data per square centimeter beats visual breathing room.
4. State is never ambiguous. Server status, scan progress, item health: always legible at a glance.
5. The interface is the tool, not a brochure for the tool.
